1.4 Product Scope

The goal of this software is to make smartphones and the internet more accessible for searching any format of file locations with desired keyword. The purpose of the application is to easily bring the file location like pdf ,doc and etc. from the given URLS in customized cluster. There is currently no such application on the market for competition. The product will be very much beneficial for reaserchers and students who are willing to get some article from any url for their education and research purpose.

2. Overall Description

This part provides a comprehensive overview of the product "File-Book" as well as an explanation of the software's features and goals. It also includes information about the product's dependencies, limitations, operating environment, user classes, and user manuals.

2.1 Product Perspective

"File Book" is the smart platform for searching any files location of any particular format from specific websites. It allows us to scrape files of various formats and search on them. After showing the search result, people can download the informations of the files. With this, people can easily access their desired files from the web and the app would be the new smart medium for this. This software will be available on both the web and the android platform.

2.2 Product Functions

The user determines how the product works. The goal of this software is to make it easy for users to search their files location with desired keyword.

2.3 User Classes and Characteristics

The user will be the primary emphasis of the Android app and web, and users will be able to use the app having to sign in. Users will be able to search for their preferred file using the voice search command. After that, they will be able to download the list of file information.

2.4 Operating Environments

The operating environment of "File Book" is listed as follows:

  • Distributed Database
  • Client/Server System
  • Operating System: any system that supports a modern web browser, android version 5.0+
  • Database Configuration: Django and SQLite
  • Platform: Web and Android

2.5 Design and Implementation Constraints

Currently, the mobile application of the product is only available for android users. To save the files that are scraped from the targeted websites and to save the scraped file in a API so that it can be retrieved and shown in the app and on the web later.

2.6 User Documentation

The product will give users with basic running instructions. Also, a short tutorial video that shows how to use the software.

2.7 Assumptions and Dependencies

The system requires a smartphone running Android 5.0 Lollipop or above with a microphone to listen to the user's voice search command. In order to  get search results from the server, the user must also have an internet connection. We used Google's Voice to Text API for our speech recognition algorithm. Users are expected to just utilize Google as a social media sign-in platform.



3. External Interface And Requirements

3.1 User Interfaces

When the user uses the service for the first time, they can use their Google account to sign in. If the user is already logged into the system, they will land on the home page.

3.2 Hardware Interfaces

Apart from a microphone, the "File Book" software does not require any more hardware.

3.3 Software Interfaces

The database will be accessed over the internet by both the mobile application and the website. Such interfaces will be provided by the operating system. Both systems will have functions for reading and writing data.

3.4 Communications Interfaces

The app will send audio to Google's Could server for analysis and will get text data from the audio. All of this takes place over HTTP queries, with Google's technology encrypting the audio and data sent.

    5. Other Nonfunctional Requirements

    5.1 Performance Requirement

    Based on the Intranet connection, each interface between a user and software must have a suitable response time.The website’s load time should not be more than one second for users. In case of crawling from website, this should not take more than 10 hours based on the number of urls in a single cluster. A notification mail will be send to user when the cluster is ready. The searching will give the result within less than 5-10 seconds.

    5.2 Safety Requirement

    Information should be safely transferred to the server, with no modifications to the data.In our product,when a user is using the product,one should always use the product with conserning safety issue.Any Urls,which is designed for not crawling and have some severe consequences can not be used.

    5.3 Security Requirement

    These features are necessary even for stand-alone systems; you don’t want anyone to have access to your sensitive data. Users personal data and cluster informations will not be accessed by anyone other than user. All the data will be stored safely.

    5.4 Software Quality Attributes

    5.4.1 Availability

    This attribute is indicative as to whether an application will execute the tasks it is assigned to perform. Availability also includes certain concepts that relate to software security, performance, integrity, reliability, dependability, and confidentiality. In addition, top-notch availability indicates that a software-driven system will repair any operating faults so that service outage periods would not exceed a specific time value.

    5.4.2 Usability Requirement

    The website’s interface has to be user-friendly and easy to use.

    5.5 Business Rules

    Based on the business models, the product will be a freemium product as this is not actually giving any file ,but this is giving the location of the certain file from where user have to collect the file by themselves. So, the product will not be able add any extra features for any kind of subcription
